圖庫的兩個檢視維度收斂設計 · 未定案

兩個獨立入口、互不打架:① 搜尋(純 tag 布林,現有)＋ ② 瀏覽(照資料夾分類,新)。瀏覽模式 = 真實資料夾樹 + 麵包屑下鑽 + 夾內再疊 tag 篩(你說的「資料夾路徑 + tag」)。資料夾樹直接讀 photo_location.rel_path 即時建,不改 schema、不依賴匯入確認過沒。

同一個圖庫、兩種找圖路徑。左右各一個 activity bar 入口,狀態各自獨立,隨時切換。

① 搜尋

by tag · 布林 AND / 排除 · 現有

不管圖放在哪個夾,用標籤跨整庫找。問「所有蔚藍檔案的笑臉」這種跨資料夾的問題。

蔚藍檔案 AND smile AND long_hair
→ 不限資料夾,命中 2,876 張

② 瀏覽

by 資料夾 · 逐層下鑽 · 新

照你原本的資料夾分類逛,「我記得放在哪個夾」時最快。進到夾裡還能再疊 tag 篩。

📁 Pixiv / 2024 這個夾裡
再篩 蔚藍檔案 + smile → 312 張

兩者用同一份資料:搜尋走 tag → photo;瀏覽走 rel_path → photo。瀏覽夾內疊 tag = 資料夾範圍 AND tag(後端只多一個路徑前綴條件)。

② 瀏覽:左側真實資料夾樹(數字=該夾含子夾的圖數),麵包屑顯示位置,主區頂的篩 tag 帶就是「夾內再疊 tag」—— 圖中正示範在 Pixiv / 2024 內加 蔚藍檔案、smile,12,990 張縮到 312 張。

瀏覽怎麼動

  • 側欄資料夾樹只展 1–2 層當常用入口(不爆長)
  • 圖牆一律遞迴顯示當前夾含子夾全部圖(Pixiv=21,194)
  • 更深的夾:主區頂「子資料夾」晶片點了往下鑽,麵包屑 +1 層,無限深

夾內 tag 怎麼加(你問的)

  • 按 + tag 跳自動完成,只列這個夾裡真的有的 tag + 各自張數
  • 選了就成一顆 chip,= 資料夾範圍 AND 該 tag
  • 後端:現有 /api/search 多帶一個 pathPrefix
  • 清掉 chip → 回到純資料夾瀏覽

邊界 / 取捨

  • 解壓忘整理的三層深夾:側欄不展,靠主區子夾晶片鑽進去即可
  • 同名子夾在不同父夾下各自獨立(跟攤平 tag 不同,見第三頁)
  • 夾內無圖 / 篩無結果 → 明確空狀態,不是空白

你不確定「現在 path→tag 是怎麼回事」。一句話:它把資料夾名攤平成標籤,跟「資料夾樹」是兩件事。同一個檔案 2024/蔚藍檔案/foo.png 在兩種維度下:

不需確認、不會合併:Pixiv 下的「蔚藍檔案」和 Twitter 下的「蔚藍檔案」是樹上兩個不同節點。瀏覽即開即用,反映你硬碟當下的真實結構。

所以兩者並存、不衝突:① 搜尋維度繼續用 path tag(跨夾查詢);② 瀏覽維度用即時資料夾樹(保留階層)。同一份 photo_location 餵兩種視圖。
